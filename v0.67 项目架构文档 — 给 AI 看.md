# 极简主页 — v0.75 项目架构文档

## 1. 项目是什么

一个纯前端浏览器主页/新标签页替代品。单 HTML + CSS + JS，无构建工具，直接用 `<script type="module">` 加载。

**核心功能：**
- 多引擎搜索（搜索引擎 + 平台搜索，可切换轨道）
- 搜索建议（JSONP 拉取，支持百度/Google/Bing 等源）
- 全屏数字时钟（毛玻璃卡片风格）
- 收藏夹面板（分区管理 + 拖拽排序 + 搜索）
- 自定义壁纸（图床 URL，横竖屏自适应）
- 明/暗主题 + 跟随系统
- 配置同步（本地 JSON 导出/导入、WebDAV、GitHub Gist）

```
├── wallpaper.js

```

**关键点：**
- `source-manager.js` 是 search-sources 和 suggest-sources 的共享工厂，两个源体系各自调用一次 `createSourceManager()`

- `sync.js` 独立运作，通过 `document.dispatchEvent(CustomEvent)` 通知其他模块更新

## 4. 数据存储方案（localStorage）

所有读写**必须经过** `utils.js` 的 `getFromStorage(key, default)` / `setToStorage(key, value)`，因为它们会做 JSON 序列化/反序列化。严禁 `localStorage.getItem` 直接读字符串。

| Key | 类型 | 存储格式 | 读写位置 |
|-----|------|---------|---------|
| `theme` | string | `"light"` \| `"dark"` \| `"auto"` | settings.js / main.js / utils.js |
| `defaultExpand` | boolean | `true` \| `false` | settings.js / layout-toggle.js |
| `layout_expanded` | boolean | `true` \| `false` | layout-toggle.js |
| `custom_sources` | array | `[{id, name, url, iconType, iconValue, iconBg, iconColor, track}, ...]` | search-sources.js |
| `current_source_id` | string | 当前选中引擎的 id | search-sources.js |
| `current_track` | string | `"engine"` \| `"platform"` | search-sources.js |
| `track_last_source` | object | `{engine: "google", platform: "github"}` | search-sources.js |
| `hidden_presets` | array | 被隐藏的预设引擎 id 列表 | search-sources.js |
| `modified_presets` | object | `{presetId: {name, url, ...}}` | search-sources.js |
| `engine_order` | object | `{engineId: orderIndex, ...}` | search-sources.js |
| `suggest_settings` | object | `{currentSourceId, customSources: [...]}` | suggest-sources.js |
| `hidden_suggest_presets` | array | 被隐藏的预设建议源 id 列表 | suggest-sources.js |
| `modified_suggest_presets` | object | 被修改的预设建议源数据 | suggest-sources.js |
| `suggest_engine_order` | object | 建议源排序 | suggest-sources.js |
| `wallpaperSettings` | object | `{autoSwitch, mode, portraitUrl, landscapeUrl}` | wallpaper.js |
| `bookmarks` | object | `{categories: [...], bookmarks: [...]}` | bookmark-data.js |
| `sync_log` | array | 同步操作日志（最近 10 条） | sync.js |
| `webdav_config` | object | `{url, filename, username, password}` | sync.js |
| `gist_config` | object | `{token, gistId}` | sync.js |
| `settings_collapse_engine` | boolean | 搜索引擎折叠状态 | settings.js |
| `settings_collapse_platform` | boolean | 平台搜索折叠状态 | settings.js |

## 5. 事件系统（CustomEvent）

模块间松耦合通信通过 `document.dispatchEvent`：

| 事件名 | 触发者 | 监听者 | 用途 |
|--------|--------|--------|------|
| `source-changed` | search.js / settings.js | main.js | 搜索引擎变更后刷新搜索栏 |
| `wallpaper-update` | sync.js | wallpaper.js | 同步导入壁纸配置后即时生效 |
| `close-all-panels` | bookmark-panel.js / settings.js | bookmark-panel.js / settings.js | 打开一个面板时关闭另一个 |
| `close-suggestions` | search.js | suggestions.js | 搜索时关闭建议下拉 |
| `open-settings` | 外部 | settings.js | 以编程方式打开设置面板指定 Tab |

## 6. 搜索引擎轨道系统

v0.67 新增双轨道设计：

- **engine 轨道**：传统搜索引擎（Google、Bing、百度、搜狗、DuckDuckGo）
- **platform 轨道**：平台内搜索（GitHub 等）

搜索栏左侧有一个轨道切换按钮（箭头图标），点按在两条轨道间切换。每条轨道独立记住上次选的源。这在 `search-sources.js` 中通过 `trackLastSource` 对象管理。

## 8. 关键模式与约定

1. **`createElement(tag, attrs, children)`** — 所有模块动态创建 DOM 的唯一方式。attrs 支持特殊 key：
   - `className` → `el.className`
   - `innerHTML` / `textContent`
   - `onXxx`（以 `on` 开头）→ `addEventListener`
   - `style`（对象）→ `Object.assign(el.style, ...)`
   - `data`（对象）→ 逐个赋给 `el.dataset`
   - boolean 属性（`checked/disabled/...`）→ 直接赋值

2. **storage 统一入口**：只能用 `getFromStorage(key, default)` 和 `setToStorage(key, value)`，它们自动 JSON 序列化。`getFromStorage` 有容错：JSON 解析失败时回退到原始字符串值。

3. **主题判断**：`applyTheme(theme)` 在 `utils.js` 中。`'auto'` 时通过 `matchMedia('prefers-color-scheme: dark')` 判断，然后把 `data-theme` 属性设在 `document.documentElement` 上。

4. **面板打开/关闭**：settings 和 bookmarks 都有关闭动画（`panelShrink` / `panelFade`），通过 `classList.add('closing')` 触发，`setTimeout` 在动画结束后移除。

5. **搜索建议是 JSONP**：通过动态插入 `<script>` 标签实现跨域请求，`window[callbackName]` 接收数据。有请求 ID 机制防止过期回调污染。

6. **壁纸背景层**：`#wallpaperLayer` 是一个 `position: fixed; z-index: -1` 的独立 div，不直接操作 body 背景。

## 10. 注意事项

- 这是一个**纯静态前端**项目，直接在浏览器打开 `index.html` 即可，不需要服务器。
- 所有数据存储在 localStorage，清除浏览器数据会丢失配置。建议定期用同步功能备份。
- 搜索建议依赖 JSONP，如果第三方 API 失效则建议不显示（不会阻塞搜索）。
- GitHub Gist 同步需要 Personal Access Token（classic），WebDAV 需要支持跨域的 WebDAV 服务。
